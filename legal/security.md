# Security Overview

## 1. Architecture
- client → API → database (role-controlled)
- access controlled via authentication + roles

---

## 2. Data Protection
- encrypted in transit
- access restricted by role

---

## 3. Responsibilities
Users must:
- secure their credentials
- restrict access to authorized persons

---

## 4. Limitations
No system is fully secure.

The platform reduces risk but does not eliminate it.

---

## 5. Incident Handling
Security issues should be reported immediately.

The platform will take reasonable corrective action.
